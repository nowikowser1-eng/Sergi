# 🚀 Prompt Kit v2.0 — Полный цикл разработки приложений на Claude Opus 4.7

> **Максимально эффективный набор промтов** для разработки, ревью и оптимизации приложений.
> Основан на [официальных best practices Anthropic](https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices) для Claude Opus 4.7 с настройкой `xhigh`.

---

## 📦 Что внутри

| № | Файл | Когда использовать | Выход |
|---|------|--------------------|-------|
| 01 | `01_requirements.md` | Есть идея/бриф, нужны формальные требования | SRS-документ, user stories, модель данных, риски |
| 02 | `02_architecture.md` | Есть требования, нужна архитектура | ADR, C4-диаграммы, структура проекта, tech decisions |
| 03 | `03_implementation.md` | Есть архитектура, нужен код | Production-grade код + тесты + setup |
| 04 | `04_review_and_fix.md` | Есть код, нужен аудит и фикс багов | Список находок по severity + патчи + тесты |
| 05 | `05_optimization.md` | Код работает, нужно ускорить/удешевить | Профилирование + приоритизированные оптимизации |
| 06 | `06_refactoring.md` | Legacy-код, техдолг | План рефакторинга безопасными шагами |
| 07 | `07_testing.md` | Нужны полные тесты | Стратегия + unit + integration + E2E |
| 08 | `08_debugging.md` | Есть баг, нужно найти root cause | Методичная диагностика с гипотезами |
| **99** | `99_api_settings.md` | Настройки API для максимальной производительности | Конфиги для Python/TS/curl |

## ⚙️ Ключевые настройки API (всегда)

```python
client.messages.create(
    model="claude-opus-4-7",
    max_tokens=64000,                    # минимум для xhigh
    thinking={"type": "adaptive"},       # модель сама решает когда думать
    output_config={"effort": "xhigh"},   # максимум интеллекта для кодинга
    messages=[{"role": "user", "content": PROMPT}],
)
```

Подробнее — в `99_api_settings.md`.

## 🧠 Принципы, зашитые во все промты

Каждый промт построен вокруг 8 принципов из официальной документации Anthropic:

1. **Явная роль** — фокусирует экспертизу модели (`<role>`)
2. **XML-структура** — устраняет неоднозначность парсинга сложных промтов
3. **Буквальность Opus 4.7** — явно указанный scope действия каждой инструкции
4. **Критерии приёмки** — каждый выход измерим и верифицируем
5. **Борьба с overengineering** — явные ограничения scope
6. **Борьба с галлюцинациями** — требование читать код перед утверждениями
7. **Параллельные tool calls** — явный запрос на параллельное чтение файлов
8. **Self-verification** — чек-лист самопроверки перед завершением

## 💡 Как применять промт

**Вариант 1 — через Claude.ai (веб/десктоп/мобильный):**
1. Открыть нужный `.md` файл
2. Скопировать блок `<system>...</system>` → вставить в поле "Add style / System prompt" (если доступно)
3. Скопировать блок `<task>` + остальное → в обычное поле сообщения
4. Заменить все `[ВСТАВЬ СЮДА: ...]` на свои данные
5. Отправить

**Вариант 2 — через API:**
```python
SYSTEM_PROMPT = open("01_requirements.md").read()  # или распарсить секцию <system>
USER_INPUT = "... твой контекст ..."

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=64000,
    thinking={"type": "adaptive"},
    output_config={"effort": "xhigh"},
    system=SYSTEM_PROMPT,
    messages=[{"role": "user", "content": USER_INPUT}],
)
```

**Вариант 3 — через Claude Code:**
Положи папку `prompts_kit/` в репозиторий как `.claude/prompts/`, вызывай нужный промт командой `/prompt 03_implementation` (если настроено).

## 📐 Структура каждого промта

Каждый файл содержит:

- **Метаданные** — для чего, когда, требования
- **`<system>`** — system prompt (роль + общие правила)
- **`<user_template>`** — шаблон пользовательского сообщения с placeholder'ами
- **Примеры использования** — реальные кейсы
- **Чек-лист после получения ответа** — как проверить качество

## ⚠️ Важные отличия v2.0 от v1.0

| Было в v1.0 | Стало в v2.0 |
|-------------|--------------|
| Один промт = одна роль | Разделены **system** и **user** части |
| Монолитные инструкции | Многоуровневые XML-теги с явными scope'ами |
| Общие фразы вроде "качественно" | Измеримые критерии и численные пороги |
| Нет примеров | Few-shot примеры там, где они критичны |
| Не учтены особенности Opus 4.7 | Добавлены: литеральность, адаптивное мышление, антипаттерны xhigh |
| 4 промта | 8 промтов + настройки API |
| Нет чек-листов самопроверки | Каждый промт заканчивается verification_checklist |
| Нет защиты от анти-паттернов | Явные блоки против overengineering, галлюцинаций, избыточного тулинга |
